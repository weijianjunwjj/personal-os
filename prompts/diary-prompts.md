# 日记整理提示词

用途：让 AI / Codex 把用户口述整理成 `personal-os` 日记条目，但不改变原始事实。

## 基本要求

- 原始输入必须完整保留。
- 事件只做简短摘要，不增加未出现的信息。
- 判断只从“我发现 / 我意识到 / 以后应该 / 应该”等明确表达中提炼。
- 没有明确判断时写“待复盘”。
- 行动只从明确行动表达中提炼。
- 没有明确行动时写“暂未形成明确行动”。
- 不鸡汤，不拔高，不编造。
- 不把日记复制到 `ai-os`。

## 输出结构

```md
## HH:mm

### 原始输入

用户原始文本

### 事件

基于原始输入的简短摘要，不编造。

### 判断

待复盘

### 行动

暂未形成明确行动

### 给未来 AI 的备注

这是一条来自日记入口的原始记录，后续可用于复盘和主题沉淀。
```

## 后续联动

写入日记后，由 `emit-event.ts` 生成机器事件，再由 `dispatch-event.ts` 分发到 `energy-os`、`ai-os` 或 `personal-os/content/career`。AI 不直接修改其他系统的正式资料。
