# 日记规则

1. 日记只存放在 `personal-os/content/diary`。
2. 每天一个 Markdown 文件。
3. 路径格式：`content/diary/YYYY/MM/YYYY-MM-DD.md`。
4. 同一天多次记录时追加，不覆盖。
5. 必须保留用户原始输入。
6. 每条记录结构包括：原始输入、事件、判断、行动、给未来 AI 的备注。
7. 不夸大。
8. 不鸡汤。
9. 不编造事实。
10. 不把日记复制到 `ai-os`。
11. 日记是人看的原始材料。
12. `event jsonl` 是系统看的机器信号。
13. 如果用户只是吐槽，也要记录为原始输入，不强行拔高。
14. 如果没有明确行动，写“暂未形成明确行动”。
15. 如果没有明确判断，写“待复盘”。

`personal-os` 是唯一原始个人数据源。其他 OS 只能接收事件沉淀出的候选数据或 draft，不能反向覆盖日记。
